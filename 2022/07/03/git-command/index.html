Git命令

发表于2022-07-03|更新于2023-03-02|学习git
|字数总计:649|阅读时长:2分钟|阅读量:

前言

当我们对代码进行了修改，但是修改完之后又想回退到以前的某一个版本，这时我们可以使用git对代码进行及其方便的本地或云端版本管理。下面介绍git的使用和一些常用命令。

本地仓库

初始化

在存放代码的文件夹中使用 $ git init 命令初始化本地仓库，完成后文件夹将新增一个隐藏文件 .git 。

添加文件至暂存区

当我们的代码大体上写好，还未修改时，我们可以先使用 $ git add [文件名] [文件名]... （添加所有文件则使用 $ git add . ）命令将代码添加到暂存区，使用 $ git status 可以查看所有文件的状态，文件名为红色，代表文件未添加至暂存区；文件名为绿色，代表文件已添加至暂存区。

提交代码至本地仓库

当我们确认代码的版本之后，可以使用命令 $ git commit -m "[提交备注]" 将暂存区中的代码提交至本地的仓库。而使用命令 $ git commit -am "[提交备注]" 可以跳过将 代码添加至暂存区 这一步操作直接保存这个版本。使用 $ git log 可以查看所有提交过的版本及信息。

回退版本

当我们需要将代码回退到之前提交过的某一个版本时，使用命令 $ git reset --hard [版本commit id] 将代码回退到该版本。

云端仓库(github)

初始化

使用命令 $ git clone [repository的url] 将github的repository克隆至本地文件夹。

接下来的 代码添加至暂存区 提交代码至本地仓库 回退版本 操作与本地仓库的操作一样。

提交代码至云端仓库

使用 $ git push origin 将代码提交至当前分支上。

分支

使用 $ git branch [分支名] 创建新的分支，新分支会继承主分支的代码，在新分支上进行的所有操作不会影响主分支（新建的分支提交代码时要使用 $ git push --set-upstream origin [分支名] 在云端绑定分支并提交）。使用 $ git branch 查看所有分支及当前所在分支，使用 $ git checkout [分支名] 切换当前分支。使用 $ git branch -D [分支名] 直接删除分支。

本地与云端连接

使用 $ git remote add origin [repository的url] 将本地仓库与云端仓库进行连接。

远程更新

使用 $ git pull 直接将远程代码更新至本地。
